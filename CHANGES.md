# Changes

## Unreleased

### Changed
- `llm_translate` now uses a two-step prompt flow for the encode direction when `LLM_TWO_STEP = True` (default). The previous single prompt asked the model to simultaneously produce cover prose and emit a JSON mapping in the same response, which produced a bimodal recovery distribution on mistral:7b-instruct (n=180 trials: half the trials emitted a 2-entry mapping with minimal rewrite, half emitted a 14+ entry mapping with full rewrite), caused qwen3 family to emit mapping JSON without any cover prose, and left phi3:mini emitting cover prose without any mapping JSON. The new flow separates the two tasks: Call 1 asks the model to emit only a JSON object of term substitutions (no prose); Call 2 provides that fixed mapping back to the model and asks for the rewrite constrained to those substitutions. Locking the mapping before the rewrite removes the model's freedom to skip either task and eliminates the 31% hallucinated-mapping rate (mapping entries that did not appear in the cover). Set `LLM_TWO_STEP = False` at the top of the LLM Backend section to revert to the one-step prompt for A/B comparison.

### Added
- Production mapping-verification gate: `verify_mapping_support` spot-checks up to 5 random `(src, tgt)` pairs from the LLM-emitted mapping against the cover prose and prints a stderr warning when fewer than 80% are supported; `--strict` flag on `encrypt` turns the warning into a hard error (non-zero exit).
- Optional polish step for the LLM backend (`--polish` flag on `encrypt`, `polish=True` kwarg on `encode_document`). After `llm_translate` returns the cover and mapping, a second LLM call asks the model to rewrite the cover for naturalness while preserving every cover-domain term exactly. The motivation is Caudle Distance reduction: eval v2 measured SCD 0.9-1.3 nats against a native brewery corpus, an order of magnitude above the Section 4.4 target of ~0.1 nats; the polish step described in that section is the path from research-grade to production-grade cover quality. A verify-and-fallback mechanism spot-checks 5 random mapping pairs against the polished cover; if fewer than 4 of 5 cover-domain targets are still present, the polish is discarded and the unpolished cover is used instead (with a one-line stderr note). The flag defaults to off to preserve backward compatibility and avoid doubling encode latency when polish is not needed.
- Evaluation v2 results: 180 trials (6 models × 3 docs × n=10) with the mapping-verification gate active. RESULTS.md is reframed around viability tiers (production / beta-consistency / beta-peak / research / unsuitable) with `P(recovery >= 0.95)` and `P(recovery < 0.30)` as the production-relevant metrics. Headline reversal vs v1: mistral:7b-instruct has the highest mean recovery (0.703) but also 13% tail risk and a 0.69 mapping-support ratio (one in three of its emitted mapping entries is hallucinated); gemma3:4b is the only model in the cohort with `P(rec < 0.30) = 0` and a 0.95 mapping-support ratio. qwen3:1.7b with `/no_think` has 0.01 mapping support across 81 sampled pairs — the thinking-mode bleed identified in v1 was not the root cause, the root cause is deeper prompt-following failure.
- Evaluation harness v2: mapping-verification gate (samples 5 random `(src, tgt)` pairs and counts how many `tgt` substrings appear in the cover, exposing the silent "hallucinated mapping" failure mode), per-model prompt-prefix dict (qwen3 family gets `/no_think` injected via `OLLAMA_PROMPT_PREFIX`), n=10 trials per (model, doc).
- `OLLAMA_PROMPT_PREFIX` environment variable: prepends the given string to every `llm_translate` prompt. Intended for model-specific directives like qwen3's `/no_think`.
- Brewery corpus expanded ~3.3x by word count and ~5x by embedding vocab (782 -> 2560 words, 106 -> 517 unique tokens), four sub-genres (audit reports, brewmaster logs, regulatory submissions, trade press, brewing process reference). Gives Caudle Distance enough samples to discriminate.

### Added
- `evaluation/` directory with a formal local-model evaluation: harness, three sensitive test documents, brewery corpus, raw per-trial JSON, and a `RESULTS.md` writeup. Headline finding: `gemma3:4b` is the most consistent local model for the LLM backend, `llama3.2:3b` has the highest peak recovery but with 1.6x the variance, and qwen3 family is unsuitable in default thinking-mode settings.

### Added
- `sanitize` and `desanitize` subcommands. Sanitize mode strips all domain-specific identities into opaque labels (`ASSET_001`, `PARAM_003`, `SYS_007`, ...) and shifts every numeric value by a constant derived from `hash(document + timestamp_salt)`. Math, structure, and relational logic are preserved; identities are gone. Intended for LLM analysis: a model can reason about vulnerability patterns, detection gaps, and remediation against the sanitized form without knowing what physical system it sees. Round-trip is exact via the embedded passphrase-protected key. Usage: `python falsecolors.py sanitize --source f.txt --passphrase X --output s.txt`.
- `measure` subcommand: computes the Caudle Distance (SCD per Definition 2.15.2) of a cover document against a native cover-domain corpus. Pure stdlib. Builds PPMI co-occurrence vectors from the corpus, scores cosine similarity over adjacent content tokens, and returns the KL divergence between the document's TAD and the corpus's TAD with Laplace smoothing. Usage: `python falsecolors.py measure --source cover.txt --corpus brewery_native.txt`. A native document scored against its own corpus returns SCD ≈ 0; an unshifted source-domain document returns a high SCD with very few adjacent pairs scored.
- `OLLAMA_TIMEOUT` environment variable controls the Ollama request timeout for `llm_translate` (default 120s). Thinking-class local models (e.g., qwen3 thinking variants) routinely exceed 120s on a standard finding document; bump to 600 with `OLLAMA_TIMEOUT=600 python falsecolors.py encrypt ...` when using such models.

### Fixed
- IdentEncoder placeholder collision past 40 unique identifiers. The pool of 40 NATO/word names was indexed with `% len(POOL)`, so the 41st unique identifier silently reused the 1st placeholder and decoding became ambiguous. Past the first cycle, placeholders are now suffixed with the cycle number (`alpha`, ..., `nectar`, `alpha2`, ..., `nectar2`, `alpha3`, ...), preserving uniqueness without bound.
